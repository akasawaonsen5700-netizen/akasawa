赤沢温泉旅館 SNS・GBP自動配信デモ

猫 × ぬる湯 × 渓流 × 静養 を軸に、アップロード → 下書き生成 → 承認 → 自動投稿までを一気通貫。

今回のデモ構成

デモは Googleスプレッドシート＋Firebase＋Netlify Functions を中核にしています。 画像/動画はWeb画面から簡単に登録し、承認キューと投稿スケジュールはSheetでも追える設計です。

ブランド統一 自動分類 AI下書き レビュー待ち判定 SNS/GBP配信
公式ブランド表現は「塩原温泉の猫と源泉長湯にほどける、静養型ウェルネスの小宿 赤沢温泉旅館」を基準に固定しています。

② Googleスプレッドシート連携

このデモは Web入力を基本にしつつ、Googleスプレッドシートを承認台帳・投稿予定表として使えるようにしています。 Sheetから直接登録したい場合は、同梱の Apps Script を貼り付けると、行追加 → Netlify Webhook送信 で同じキューに入ります。

ファイルそのものは Firebase Storage に保存します。理由は、画像/動画の扱いがGoogle Sheets単体より安定し、Netlifyとの相性が良いからです。
  • Sheet列: mediaUrl / ownerComment / location / catName / simpleTag / publishAt / channels
  • Apps Scriptで「新規行をWebhook送信」
  • 承認後は publish-approved 関数が各配信先WebhookへPOST
